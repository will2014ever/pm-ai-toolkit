Deep Dive 1

PCR & Lessons-Learned Assistant

A source-grounded post-mortem workflow is one of the cleanest AI starting points because the input set is known and the output is easy to review.

Lifecycle: Post MP (Closing) PMI: Integration, Quality, Communications AI Agent: NotebookLM + LogiQ

Why This Matters

PCR preparation is one of the most time-consuming PM tasks. It requires pulling data from dozens of documents, reconstructing timelines, and synthesizing months of decisions into a coherent narrative. AI reduces this from days to hours.

Recommended Workflow

Gather the Project Source Set

Collect all key artifacts: PRD, GPS, SW spec, milestone decks, key meeting minutes, issue history, change history, test reports, and risk logs. This becomes the "source of truth" for the PCR. Be thorough — NotebookLM can only work with what you give it.

Typical: 15-30 documents per project

Consolidate & Convert Files

Use LogiQ or a simple script to convert files to reviewable formats (PDF, plain text). Remove duplicates and clearly outdated drafts. Organize by project phase for easier synthesis.

Tools: LogiQ for quick conversion, batch PDF tools

Load into NotebookLM for Grounded Synthesis

Upload the curated document set into NotebookLM. Use it for: timeline reconstruction, key decision extraction, risk outcome tracking, and draft summary generation. Every output is traceable to a source document.

Tools: NotebookLM (primary), LogiQ (supplementary analysis)

Review, Correct, and Finalize

PM reviews the AI-generated draft for accuracy. Add human judgment: stakeholder sentiment, political context, strategic lessons that aren't captured in documents. Convert into the final PCR narrative and deck format.

Output: Final PCR deck + lessons-learned database entry

Prompt Templates

Based on all uploaded project documents, reconstruct the project timeline from kickoff to launch. Include: 1. Key milestones and actual dates 2. Major scope changes with dates and rationale 3. Significant risk events and how they were resolved 4. Schedule variances (planned vs. actual) Cite the source document for each event.
Analyze all project documents and extract: 1. Top 5 things that went well (with evidence) 2. Top 5 things that should improve (with evidence) 3. Decisions that had the biggest positive impact 4. Decisions that had the biggest negative impact 5. Recurring themes across milestone reviews Group by category: Technical, Process, Communication, Resource, Vendor/External
Using these findings from the project review: [paste NotebookLM synthesis] Draft a 1-page executive PCR summary with: - Project overview (scope, timeline, team) - Key results vs. original targets - Top 3 successes and top 3 improvement areas - Root cause analysis for major variances - Actionable recommendations for future projects Tone: factual, constructive, forward-looking.
PM value: Faster PCR prep, clearer storyline, easier knowledge reuse across projects. What used to take 2-3 days now takes 3-4 hours of focused work.
Tip: Build a standard source checklist for your team. Every project should collect the same set of artifacts. This makes each PCR faster because the AI workflow is repeatable.
