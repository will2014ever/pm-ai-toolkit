Deep Dive 6

Vibe Coding & Rapid MVP

PMs build tools, dashboards, and landing pages directly with AI coding assistants. From idea to working prototype in hours, not weeks. No engineering ticket needed.

Lifecycle: All stages PMI: Integration, Quality, Communications AI Agent: Claude Code (Build agent)

What is Vibe Coding?

Vibe coding means describing what you want in natural language and letting AI write the code. The PM focuses on what the tool should do, not how to code it. Claude Code handles HTML, CSS, JavaScript, data processing, and logic.

Recommended Workflow

Define the Problem

What PM task is painful or slow? What tool would make it better? Write a clear 2-3 sentence description of what the tool should do, who uses it, and what the output looks like.

Input: Problem statement in natural language

Open Claude Code and Describe

Start a Claude Code session. Describe the tool you want: layout, inputs, calculations, and desired output. Be specific about data fields, formulas, and visual structure. AI generates the first working version.

Tool: Claude Code with Claude Opus

Review the First Output

Open the generated file in a browser. Check: Does it look right? Do the calculations match? Is the layout intuitive? Note what needs changing — field names, layout, missing features, calculation errors.

Review: Open in browser, test with real data

Iterate with Natural Language

Tell Claude Code what to change: "add a Packaging section", "make the total row blue and bold", "add a print button that hides the form". Each iteration refines the tool. Typically 3-5 rounds to reach a usable MVP.

Iteration: Conversational refinement

Deploy and Share

The output is a self-contained HTML file. Share via email, Slack, or Drive. No server needed. Colleagues open it in any browser. For more complex tools, deploy on an internal site.

Output: Single HTML file, shareable instantly

What PMs Can Build

Product Cost Calculator

Convert Excel cost models into web interfaces. Live calculations, printable summaries, scenario comparison.

Risk Mitigation Tracker

Visual risk register with heat maps, status tracking, and exportable reports.

Project Dashboard

Single-page status view with RAG indicators, timeline visualization, and key metrics.

Issue Resolution Log

Structured issue capture with root-cause fields, owner assignment, and status flow.

Teardown Checklist

Step-by-step teardown workflow with photo capture fields and structured observations.

SW/FW Architecture Viewer

Upload source code and get an AI-generated architecture overview. Helps PMs understand technical design faster.

Meeting Action Tracker

Post-meeting tool: paste transcript, AI extracts actions, owners, and deadlines into a trackable list.

Team Resource Planner

Visual capacity planner showing team allocation across projects with over-allocation warnings.

Example Prompts

Create a single-page HTML cost calculator. Layout: 3-column card grid. Column 1: EE Components (17 line items) Column 2: ME Components (8 line items) Column 3: MFG Parameters + Additional Costs Bottom: Live cost summary showing subtotals, material total, conversion total, and grand total. All calculations happen on every keystroke. Add a Print button that shows only the summary.
Build an interactive risk register as a single HTML page. Features: - Table with: Risk ID, Description, Category, Probability (1-5), Impact (1-5), Score (auto-calc), Owner, Mitigation, Status - Color-code rows by risk score (red/amber/green) - Add/edit/delete rows - Sort by any column - Heat map visualization (probability vs impact grid) - Export to CSV button
Create a project status landing page. Sections: 1. Project header (name, PM, dates, phase) 2. RAG status indicators (Overall, Schedule, Budget, Quality, Risk) - clickable to change 3. Key metrics: % complete, budget burn, open issues 4. Milestone timeline (horizontal bar) 5. Top 5 risks with mitigation status 6. This week / next week highlights Make it printable. Use blue accent color. Single self-contained HTML file.
PM value: Faster discovery, better design framing, and stronger early stakeholder alignment. PMs who vibe-code become 10x more self-sufficient — prototyping ideas instead of writing requirements documents and waiting for dev capacity.
Tip: Start with something you personally find tedious (cost tracking, status formatting, data consolidation). The best PM tools are the ones that eliminate your own pain points first.
