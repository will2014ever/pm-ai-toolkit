Deep Dive 4

Fine-Tune Milestone Visuals with AI

AI-generated visuals help PMs explain product issues more clearly in milestone decks, especially when the root cause is physical and difficult to describe with words alone.

Lifecycle: Gate 1 – Gate X (Refine → Launch) PMI: Communications, Quality, Stakeholder AI Agent: Nano Banana (Research agent)

Real Example: Switch Flash Issue

A product (Kodiak) showed a main-click concern. The root cause pointed to flash on the switch. Reference photos and engineering notes defined what the switch should look like in qualified vs. failed conditions. AI generated side-by-side comparison visuals that made the subtle physical differences easy to see and explain in the milestone deck.

Recommended Workflow

Frame the Issue Clearly

Start by writing a clear problem statement: What is the issue? What is the root cause? What physical or technical difference needs to be shown? The clearer your brief, the better the AI output.

Input: Problem statement + root cause description

Provide Reference Material

Use reference photos, engineering notes, sketches, or diagrams to define the "good" vs. "bad" conditions. AI needs visual references to generate accurate comparisons.

Input: Photos, sketches, CAD screenshots, engineering notes

Generate Comparison Visuals

Ask Nano Banana to generate side-by-side visuals showing the qualified vs. failed conditions. Request clear labels, annotations, and highlighted difference areas.

Tool: Nano Banana (AI image generation)

Integrate into Milestone Deck

Use the generated visuals to build the root-cause story in the deck. Side-by-side comparisons help non-technical stakeholders understand the defect mechanism quickly.

Output: Visual slides for milestone/gate review

Review with Engineering Team

PM and engineering review generated visuals. Correct any inaccuracies, keep only the versions that improve understanding. AI visuals supplement — not replace — actual engineering data.

Review: Accuracy check with domain experts

Prompt Templates

Generate a side-by-side comparison image showing: LEFT: Qualified switch condition - Clean contact surface, no flash visible - Proper actuation height RIGHT: Failed switch condition - Visible flash/burr on contact area - Flash interfering with actuation mechanism Label both sides clearly. Use arrows to highlight the key difference. Technical illustration style, clean and professional for a milestone deck.
Create an annotated technical illustration of [component]: 1. Show the component in assembled context 2. Highlight the defect area with a red circle/arrow 3. Add a zoomed inset showing the defect detail 4. Include text labels for key dimensions/features 5. Use clean engineering-drawing style Purpose: Explain root cause to non-technical stakeholders in a gate review presentation.
PM value: Clearer storytelling, faster stakeholder understanding, and stronger visual support for engineering decisions. Reduces the "I don't understand the issue" friction in gate reviews.
Tip: Always show AI-generated visuals to the engineering team before presenting. Accuracy matters more than aesthetics. Use AI visuals as communication aids, not as engineering evidence.
